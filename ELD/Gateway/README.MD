# CAN Logger ELD Prototype API Specification

## Overview

This document defines the communication interface between the **CAN Logger ELD** and the mobile application.

The ELD acts as a Wi-Fi Access Point (AP). The mobile application connects directly to the ELD and communicates using HTTP requests over the local Wi-Fi network.
---

# Network Configuration

| Parameter | Value           |
| --------- | --------------- |
| Port      | `80`            |
| Protocol  | HTTP/1.1        |

---

# General API Rules

* All requests use HTTP.
* The ELD does **not** maintain persistent sessions.
* Every request is independent.
* All responses return HTTP status codes.

---

# API Endpoints

---

## Driver Login

### POST

```
POST /login
```

### Purpose

Notify the ELD that the driver has successfully authenticated using the mobile application.

---

### Request

```json
{
    "username":"driver01",
    "authenticated":true
}
```

---

### Response

```http
HTTP/1.1 200 OK
```

---

## Driver Logout

### POST

```
POST /logout
```

### Purpose

Ends the current driver's session.

---

Response

```http
HTTP/1.1 200 OK
```

---

# Driver Information

### POST

```
POST /driver
```

### Purpose

Registers or updates the current driver's information.

---

Request

```json
{
    "firstName":"John",
    "lastName":"Smith",
    "username":"driver01",
    "licenseNumber":"A12345678",
    "licenseState":"CO"
}
```

# Duty Status

### POST

```
POST /duty
```

### Purpose

Changes the driver's duty status.

---

Request

```json
{
    "status":"DRIVING"
}
```

Supported values

```
OFF_DUTY

SLEEPER

DRIVING

ON_DUTY
```

### POST

```
POST /timezone
```

Example

```json

    "utcOffset":-7
}
```

# Current Status

### GET

```
GET /status
```

Returns current ELD status.

Example response

```json
{
    "gps":true,
    "can":true,
    "driverLoggedIn":true,
    "currentDuty":"DRIVING",
    "wifiClients":1
}
```

# Retrieve RODS

### GET

```
GET /rods
```

Returns the driver's Record of Duty Status for the requested period.

Future versions may support query parameters.

Example

```
GET /rods?days=8
```

---

# Retrieve Events

### GET

```
GET /events
```

Returns ELD event records.

---

# Retrieve Diagnostics

### GET

```
GET /diagnostics
```
